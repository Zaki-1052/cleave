# 2026-03-27 — Phase 4.1: Peak Calling Pipeline Module

## What Was Done

- Created `backend/pipelines/peak_calling.py` (~800 lines) — full PeakCallingStage with validate, run (real mode), mock_run, and generate_methods_text
- Supports all 5 peak caller modes: MACS2 narrow/broad, SICER2, SEACR stringent/relaxed
- Fragment size filter (<120bp) via `filter_below.awk` as default-ON pre-step
- IgG control BAM passed as MACS2 `-c` flag; filtered IgG cached across reactions
- SEACR chain matches lab exactly: MACS2 bdg (no --SPMR) → change.bdg.py → SEACR_1.1.sh
- FRiP calculation via bedtools intersect / samtools view -c
- HOMER annotation (non-fatal on failure)
- Post-peak blacklist subtraction via bedtools subtract
- Mock mode loads canned CUTANA Cloud CSVs (tab-separated), creates stub BED/annotation files
- Added `peak_calling_methods()` to `backend/pipelines/methods_text.py`
- Added `PeakCallingReactionMetrics`, `TopCalledPeak`, `PeakCallingQCReport` to `backend/schemas/qc_report.py`
- Registered `"peak_calling"` in `backend/pipelines/__init__.py` dispatcher
- Created `backend/tests/test_peak_calling_pipeline.py` — 49 tests (validation, mock run, methods text, helpers, CSV writers, schemas, constants)

## Decisions Made

- MACS2 default q-value = 0.01 (lab standard, not CUTANA's 0.05)
- IgG passed as `-c` flag (CUTANA Cloud pattern, not in lab scripts)
- `bedtools sort` replaces `sort-bed` (BEDOPS) — already a dependency
- HOMER failure is non-fatal warning (core pipeline + FRiP still complete)
- Canned CUTANA CSVs are tab-separated; our written CSVs are comma-separated (matching alignment pattern)
- Helper functions duplicated from alignment.py (each module self-contained, matching existing pattern)
- New `_run_triple_pipe()` helper for the 3-command fragment filter pipe
