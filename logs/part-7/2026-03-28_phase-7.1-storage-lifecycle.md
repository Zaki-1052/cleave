# 2026-03-28 ‚Äî Phase 7.1: Storage Lifecycle Management

## What Was Done

- **Cleanup service** (`backend/services/cleanup_service.py`): Deletes expired pipeline logs (30-day retention) and stale tus staging files (48h retention). Atomic `storage_bytes` decrements grouped by experiment/project.
- **Worker integration**: Periodic `_maybe_run_cleanup()` in the main loop, timestamp-gated by `CLEANUP_INTERVAL_HOURS` (default 24h). Errors logged, never crash the worker.
- **Deletion disk cleanup fix**: `delete_experiment` and `delete_project` now `shutil.rmtree()` disk directories and decrement project `storage_bytes`. Previously only DB records were cascade-deleted ‚Äî files were orphaned.
- **Admin API** (`backend/routers/admin.py`): `POST /admin/cleanup` (superuser manual trigger), `GET /admin/storage-info` (quota + disk usage for frontend gauges).
- **Frontend StorageGauge** (`StorageGauge.tsx`): Progress bar with green/amber/red thresholds when `STORAGE_QUOTA_BYTES > 0`, text-only fallback when 0. Integrated into project sidebar and experiment description tab.
- **Config**: 5 new settings ‚Äî `CLEANUP_ENABLED`, `CLEANUP_INTERVAL_HOURS`, `LOG_RETENTION_DAYS`, `STORAGE_QUOTA_BYTES`, `TUS_STAGING_RETENTION_HOURS`.
- **Tests**: 11 new tests. 384 total passing. Ruff, tsc, npm build all clean.

## Decisions Made

- **Intermediate BAMs already cleaned**: The alignment pipeline deletes all intermediates during execution ‚Äî no 7-day retention policy needed. Only pipeline logs need periodic cleanup.
- **No Alembic migration**: All retention logic uses existing `completed_at`, `file_category`, `storage_bytes` columns.
- **No disk scanning for orphans**: Deferred ‚Äî only DB-tracked `job_outputs` with `file_category="log"` are cleaned. Orphan scanning is complex and not in 7.1 scope.
- **Global quota only**: `STORAGE_QUOTA_BYTES` is a global config, not per-project DB column. Sufficient for single-EC2 deployment.

## Post-Review Hardening

- **Path traversal guard** added to `cleanup_expired_logs()`: resolves `abs_path` and validates it falls within `STORAGE_ROOT` before `unlink()`, matching the existing pattern in `routers/files.py`. Skips + warns on any path outside the root.
