## Decisions Made

- **No external drag-and-drop library** — native HTML5 `dragenter/dragleave/dragover/drop` events are sufficient
- **Single aggregate progress bar** — Axios `onUploadProgress` gives one stream for the whole multipart POST; per-file progress would require tus (deferred)
- **Client-side extension filtering** on drop/pick — rejects non-FASTQ files before staging, but full validation (alphanumeric start, R1/R2) deferred to backend for single source of truth
- **Upload zone collapses after success** — `onUploadComplete` closes the zone so user sees their files in the table immediately
- **Delete button uses red override** on primary Button — `className="bg-red-600 hover:bg-red-700"` for destructive action
- **FASTQC and Total Reads columns** are placeholder stubs (show "—") — wired in Phase 2.3/2.4
