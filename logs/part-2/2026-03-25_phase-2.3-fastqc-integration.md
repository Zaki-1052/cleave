## Decisions Made

- **Background task, not blocking**: FastQC runs via FastAPI `BackgroundTasks` ‚Äî upload returns immediately with `totalReads=null`, frontend polls until populated
- **New DB column over convention-based paths**: `fastqc_report_path` is explicit and queryable vs deriving from filename
- **Own DB session in background task**: Background task uses `async_session_factory()` directly since request session is closed
- **Scalar extraction before background dispatch**: Extract `fastq_id`, `file_path`, `filename` into plain dicts to avoid SQLAlchemy lazy-load issues after session close
- **Docker sample dir resolution**: `_resolve_sample_dir()` checks `/cutana/fastqc` (Docker mount) then falls back to relative project path (local dev)
