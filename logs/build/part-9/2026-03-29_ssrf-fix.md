# SSRF Bypass Fix — 2026-03-29

## What was done

- Ran `/security-review` on the FTP/SFTP server import feature (commit `9ad730f`)
- Identified SSRF bypass: `0.0.0.0/8` missing from `_BLOCKED_NETWORKS` and IPv6-mapped IPv4 addresses (`::ffff:127.0.0.1`) evading IPv4 blocklist checks
- Added `0.0.0.0/8` to the SSRF blocklist
- Added IPv6-mapped IPv4 unwrapping in `_validate_host()` so addresses like `::ffff:127.0.0.1` are checked against IPv4 blocked ranges
- Added 3 new tests: `test_block_zero_address`, `test_block_ipv6_mapped_loopback`, `test_block_ipv6_mapped_metadata`
- Updated landing page test count: 463 → 466

## Decisions made

- Only the high-confidence (8+/10) finding was actioned; lower-confidence findings (DNS rebinding TOCTOU at 4/10, `known_hosts=None` at 4/10, etc.) were filtered out as hardening concerns rather than concrete vulnerabilities given the lab-internal deployment context
