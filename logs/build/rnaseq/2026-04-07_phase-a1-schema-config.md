# Phase A.1 — RNA-seq Schema + Config Changes

**Date**: 2026-04-07

## What was done

- Added `RNA-seq` as third assay type (`AssayType` enum + `AssayTypeValue` Literal)
- Added 4 nullable columns to `reactions` table: `treatment`, `timepoint`, `genotype`, `replicate_number`
- Updated `ReactionCreate`, `ReactionRead`, `ReactionUpdate` Pydantic schemas with new fields
- Added CSV header mappings, template headers, and `replicate_number` integer parsing in `reaction_service.py`
- Added `STAR_INDEX_DIR`, `SALMON_INDEX_DIR`, `GENCODE_GTF_DIR` config settings
- Created Alembic migration `be082d72cc1c` (4 nullable columns on reactions)
- Updated frontend `Reaction` / payload types and added `'RNA-seq'` to `ASSAY_TYPES` constant
- Fixed pre-existing bug: FastQC notification insert now wrapped in try/except (matching auto-pipeline pattern)
- Fixed pre-existing lint: removed unused `Path` import in `test_local_import.py`

## Decisions made

- New reaction columns are all nullable — backwards-compatible, CUT&RUN/CUT&Tag reactions ignore them
- No new validators on `treatment`/`timepoint`/`genotype` — free-form text fields
- Stripped spurious `saved_servers` nullable drift from autogenerated migration
- FastQC notification failure: try/except is correct fix (matches convention in same function, prevents auto-pipeline hook from being skipped)
