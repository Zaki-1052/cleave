# 2026-03-28 — Phase 7.3: Experiment History / Audit Log

## What Was Done

- **New `experiment_events` table** with Alembic migration (`4f02b80e7f9b`): id, experiment_id (FK CASCADE), user_id (FK SET NULL), action (String 50), resource_type, resource_id, detail, created_at.
- **Event service** (`services/event_service.py`): `log_event()` (transactional, no commit), `log_event_standalone()` (worker sessions), `list_events()` (paginated, auth-checked).
- **API endpoint**: `GET /api/v1/experiments/{id}/history?page=1&perPage=25` added to experiments router. Returns `PaginatedResponse[ExperimentEventRead]`.
- **10 event types** wired at all mutation points:
  - `fastq_uploaded` (tus + multipart), `fastq_deleted` — in `tus_upload.py`, `fastq_service.py`
  - `reaction_created`, `reactions_imported`, `reaction_updated`, `reaction_deleted` — in `reaction_service.py`
  - `job_launched` — in `job_service.py`
  - `job_completed`, `job_failed` — in `worker.py` via `log_event_standalone()`
  - `metadata_updated` — in `experiment_service.py`
- **Frontend**: `experimentEvents.ts` API module, `useExperimentHistory.ts` hook, `HistoryTab.tsx` with DataTable (Date, User, Action with color-coding, Details).
- **Tests**: 9 new tests in `test_experiment_events.py`. 77 existing related tests confirmed passing. All linters clean.

## Decisions Made

- **New table over deriving from existing data**: reactions and metadata changes have no audit trail; a dedicated events table is the only option that covers all action types.
- **`log_event()` does NOT commit**: event is flushed atomically with the parent operation. Worker uses standalone variant with its own session.
- **Action is a plain string, not an enum**: new event types can be added without migrations.
- **Not logged**: experiment creation (redundant with `created_at`/`created_by`), experiment deletion (CASCADE deletes events), job queued-to-running (internal state, not user-visible).

## Open Items

- Worker `job_completed`/`job_failed` events not yet covered by integration tests (would need `patch_worker_sessions` fixture + mock pipeline run).
- No SSE invalidation of history query cache — history tab requires manual refresh or re-navigation. Low priority at lab scale.
