# 2026-03-24 — Phase 1.3: Project CRUD Backend + Frontend

## What was done

### Backend (2 files)
- Added `UserBrief` schema to `schemas/project.py` and nested it in `MemberRead` so the member list API returns user names/emails
- Added `selectinload(ProjectMember.user)` to `list_members`, `add_member`, `update_member_role` in `services/project_service.py` to eagerly load user data (avoids async `MissingGreenlet`)

### Frontend (7 files, 2 new)
- Added `MemberUser` and `Member` TypeScript interfaces to `api/types.ts`

- Added `useMembers()` hook to `hooks/useProjects.ts`
- Created `hooks/useExperiments.ts` with `useExperiments()` and `useExperiment()` hooks
- Created `components/projects/CreateProjectModal.tsx` — name + description form using existing Modal/Input/Button
- Updated `pages/HomePage.tsx` — removed hardcoded StatusBadge, added storage size to cards, wired Create Project modal
- Updated `pages/ProjectDetailPage.tsx` — wired member list sidebar (teal avatars, initials, role labels), wired experiments DataTable with columns and empty state

## Decisions made
- Used nested `UserBrief` in `MemberRead` (not flat fields) because Pydantic `from_attributes=True` can't flatten ORM relationships without custom validators
- Used `selectinload` over `joinedload` — cleaner for composite-PK relationships in async SQLAlchemy
- Used raw `<textarea>` in CreateProjectModal instead of creating a Textarea component (KISS — single use case)
- Skipped "Last Job" column in experiments table (requires analysis_jobs data from Phase 3)
- Skipped member avatars on HomePage project cards (would require N+1 queries; deferred)
