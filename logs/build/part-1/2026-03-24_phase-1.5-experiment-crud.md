# 2026-03-24 — Phase 1.5: Experiment CRUD

## What was done

### Backend (3 files)
- Tightened `ExperimentCreate.assay_type` and `ExperimentUpdate.assay_type` to `Literal["CUT&RUN", "CUT&Tag"]` in `schemas/experiment.py` — Pydantic auto-rejects invalid assay types with 422
- Added `creator: UserBrief | None` to `ExperimentRead` — eagerly loaded via `selectinload(Experiment.creator)` in all service queries
- Added membership checks to `create_experiment` (admin/contributor only, returns None for 403), `update_experiment` (added `user_id` param, joins on ProjectMember), `delete_experiment` (added `user_id` param, returns bool) in `services/experiment_service.py`
- Added Query aliases (`alias="projectId"`, `alias="perPage"`) to router params so camelCase query params from the frontend are accepted
- Wired `user_id` to all service calls in router, added 403 handling on create

### Frontend (7 files modified, 1 new)
- Added `creator: MemberUser | null` to `Experiment` type in `api/types.ts`
- Extracted `getDisplayName()` and `getInitials()` to `lib/utils.ts` (shared by ProjectDetailPage and DescriptionTab)
- Added `useCreateExperiment()` and `useDeleteExperiment()` mutation hooks to `hooks/useExperiments.ts`
- Created `components/experiments/CreateExperimentModal.tsx` — name (100-char limit with counter), assay type dropdown, description textarea
- Wired "Create Experiment" button in `ProjectDetailPage.tsx` to open modal, navigate to experiment on creation, added "Last Job" column (static "None")
- Wired `ExperimentView.tsx` to fetch real experiment data, show name + status badge in header, pass experiment via Outlet context
- Wired `DescriptionTab.tsx` to show real metadata (ID, creator name, date, status, size) from Outlet context

### Tests (1 file)
- Implemented 10 test cases in `test_experiments.py`: create success, name too long (422), invalid assay type (422), nonmember create (403), list by project, get detail with creator, update, update nonmember (404), delete, viewer delete (404)
- All 31 tests pass (13 auth + 10 experiments + 8 projects)

## Decisions made
- Used regular `Modal` instead of `WizardModal` for experiment creation (only step 1 needed; convert to wizard in Phase 2)
- Added Query `alias` params to experiments router to bridge FastAPI snake_case params with frontend camelCase
- Used `useOutletContext` to share experiment data from ExperimentView to child tab components
- Viewers cannot create/update/delete experiments (admin + contributor only)
