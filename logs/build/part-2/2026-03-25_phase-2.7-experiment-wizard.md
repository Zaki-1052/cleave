## Decisions made

- Experiment is created via API on Step 1 "Next" — both FileUploadZone and ReactionsEditor require a real `experimentId`, so deferring creation would require reworking those components
- Going Back to Step 1 and re-advancing calls `updateExperiment` instead of creating a duplicate
- Wizard uses `max-w-6xl` to accommodate ReactionsEditor's wide DataTable
- Step 3 footer has "Save" (close without navigating) and "Update Experiment" (navigate to experiment view)
- Canceling after Step 1 leaves an empty "new" experiment — acceptable per CUTANA Cloud behavior
