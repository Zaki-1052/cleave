# Phase 2.6 — Reactions Frontend

**Date**: 2026-03-25

## What was done

- Built the complete Reactions Frontend wiring up all 8 backend endpoints from Phase 2.5
- Created API module (`api/reactions.ts`) with 8 functions: CRUD, CSV import, template download, prefix detection
- Created TanStack Query hooks (`hooks/useReactions.ts`) with 2 queries + 5 mutations following `useFastqs.ts` pattern
- Built `CsvUploadZone` component: drag-drop CSV upload with success/warning/error banners + Download Template link
- Built `ReactionFormModal`: add/edit form with all 16 fields, spike-in target dependency, collapsible "More Fields" section
- Built `ReactionsEditor`: reusable editor combining CSV upload + OR divider + DataTable with R1/R2 checkmarks + Customize Columns + CRUD actions
- Replaced `ReactionsTab.tsx` stub with full implementation: read-only DataTable + Edit button opening editor modal
- Extended `Reaction` type in `api/types.ts` with 9 missing optional metadata fields; added `PrefixInfo`, `CsvImportResponse`, `ReactionCreatePayload`, `ReactionUpdatePayload`
- Added `CUTANA_SPIKE_IN_OPTIONS` and `CUTANA_SPIKE_IN_TARGETS` (16 PTMs) to `constants.ts`
- Added optional `className` prop to `Modal` component for width override (backwards-compatible)
- `npx tsc --noEmit` passes cleanly

## Decisions made

- `ReactionsEditor` designed as standalone reusable component for embedding in experiment wizard step 3 (Phase 2.7)
- R1/R2 checkmarks cross-reference reactions against `/prefixes` endpoint via `useMemo` lookup map
- Customize Columns uses checkbox dropdown toggling 12 optional columns dynamically in the DataTable
- CSV upload is immediate (no staging) since CSVs are small — simpler than the FASTQ FileUploadZone pattern
- Read-only tab view + modal editor pattern (vs inline editing) for simplicity while matching CUTANA Cloud UX
- Default `perPage=100` for reactions (experiments typically have 5-10 reactions)
